# Домашнее задание: Асинхронное скачивание и парсинг данных с биржи

---

> __Необходимо организовать асинхронное скачивание файлов с сайта биржи Spimex, последующий парсинг и загрузку данных в PostgreSQL. Для подключения к БД необходимо использовать асинхронные сессии. Также требуется произвести сравнительную оценку скорости выполнения синхронного и асинхронного кода.__

### Скачивание файлов с биржи Spimex

> Источник данных: https://spimex.com/markets/oil_products/trades/results/

#### Требования к скачиванию:
* Реализовать асинхронное скачивание файлов с сайта биржи
* Использовать aiohttp или httpx для асинхронных HTTP запросов
* Обрабатывать ошибки соединения и таймауты
* Логировать процесс скачивания (количество файлов, прогресс, ошибки)
* Сохранять файлы в локальную директорию с понятной структурой
* Поддерживать докачку при обрыве соединения (опционально)

### Парсинг скачанных файлов
#### Требования к парсингу:
* Распознать формат файлов (Excel, CSV, PDF, etc.)
* Использовать соответствующие библиотеки: pandas, openpyxl, xlrd
* Извлечь структурированные данные из файлов
* Обрабатывать ошибки парсинга и валидировать данные
* Создать дата-классы или Pydantic модели для валидации
* Логировать количество обработанных записей и ошибки

### Асинхронная загрузка данных в PostgreSQL
#### Требования к БД:
* Использовать PostgreSQL для хранения данных
* Применять асинхронный драйвер: asyncpg или psycopg с async
* Спроектировать оптимальную схему БД под типы данных биржи
* Создать таблицы с правильными индексами и ограничениями
* Использовать асинхронные сессии для подключения
* Реализовать batch вставку для оптимизации производительности
* Обрабатывать транзакции и rollbacks при ошибках

### Сравнительная оценка производительности
#### Требования к сравнению:
* Реализовать синхронную версию того же скрипта
* Замерить время выполнения для синхронного и асинхронного подходов
* Использовать одинаковый набор данных для честного сравнения
* Измерять общее время и время каждого этапа (скачивание, парсинг, загрузка)
* Зафиксировать количество записей, сохраненных в БД
* Создать таблицу/график с результатами сравнения
