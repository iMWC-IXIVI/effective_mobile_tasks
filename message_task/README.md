# Задание 4. Проектирование обработки сообщений

---

- Какой контекст:

> В системе есть JsonMessage из разных источников, например, Telegram, Mattermost и Slack. Каждое JsonMessage содержит источник (type) и тело (payload) в виде словаря. Чтобы удобно работать с сообщениями внутри системы, их нужно привести к унифицированному виду ParsedMessage

- Что нужно сделать:

Требуется спроектировать систему для преобразования сообщений из разных источников (Telegram, Mattermost, Slack) в единый формат. Система должна включать: исходный класс сообщения (с типом и данными), унифицированное представление и фабрику парсеров, выбирающую подходящий обработчик для каждого типа сообщения.
